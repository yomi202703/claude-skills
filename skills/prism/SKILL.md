# prism
## 設計原則

- 発散専用。1つの判定・答え・推奨へ丸めない。価値は framing の散らばりそのもので、目どうしが衝突したらそれを成果として残す。

## 入力 — 対象を1つ固定する

prism は対象ありき。対象が曖昧なら先に1つへ絞る。対象は何でもよい。

## 目の選び方 — reference/lenses.md から

解釈の角度は reference/lenses.mdから5個以上選択。

## 各目の読み + 軽い grounding

各目について:

1. その目の角度で対象を読み、「この目で見ると何が見えるか」を書く。
2. その読みが空想でないかだけ、WebSearch を1本程度撃って最小限の足場（source 1つ）を付ける。
3. grounding が空振りした目は、捏造で埋めず「足場なし＝この読みは未確認の解釈」と明示して残す（読み自体は消さない）。

## 横断 — 発散の収穫

目をまたいで対象を眺め直し、次を1ブロックで出す（収束させず、衝突を残す）:

- 目によって対象がどう違って見えたか（同じ対象の像のズレ）。
- 目どうしの衝突点（ある目では強み、別の目では弱み、など）。これは消さない — 衝突の存在が成果。
- どの目がその対象に最も刺さったか（刺さり方の濃淡それ自体が信号）。

## 出力 — framing スペクトル

```
## prism: <対象>  目: <かけた目>

### 👁 <目の名前> で読むと
- 見え方: <この目で対象がどう見えるか>
- 足場: <source 1つ / 「足場なし＝未確認の解釈」>
（目ごとに繰り返し。件数・分量は揃えなくてよい — 刺さる目ほど厚くなる＝信号）

### 🔀 横断（目どうしのズレ・衝突）
- <像のズレ・衝突点・刺さりの濃淡>

### 取得失敗
<grounding で落ちた源があれば一行ずつ。無ければ「なし」>
```
