# 作業報告書の生成

指定日の作業を、提出用 xlsx フォーマット(template.xlsx)で生成する。本文は git コミットと`成果物/decisions` を一次情報として読み、平易な日本語の散文に書き起こす。

## 手順

1. 対象日を確定する。ユーザー指定がなければ聞く(例「6/19」→ その年の 2026-06-19)。

2. 作業時間を必ず聞く。開始 / 終了 / 休憩(分) の3点。 注意: 6時間超勤務時は60分以上の休憩が必要(テンプレート記載の出勤管理規則 第5条)。 逸脱していそうなら一言指摘する。

3. その日の作業内容を集める。一次情報を自分で読むこと(要約に頼らない)。 重要: コミットは当日ゼロのこともある(未コミットのまま進む日がある)。 コミット起点だと取りこぼすので、経緯記録(decisions)を主・コミットを補助とする。 各リポジトリ(pipeline_13_音声判定統合 / factcheck / リポジトリ直下の単発ドキュメント等)を横断して拾う:
   - 主: `成果物/decisions/`(モジュール別) と `成果物/decisions.md`、`STATUS.md` の当日(対象日)エントリ。 `grep -rn "<対象日 YYYY-MM-DD>" --include="*.md" .` で当日エントリを横断検索すると速い。
   - 補助(コミットがある日): `git log --since="<日> 00:00" --until="<翌日> 00:00" --pretty=format:"%h %ad%n%s%n%b%n---" --date=format:"%Y-%m-%d %H:%M"`
   - 未コミット分の裏取り: `git status --short` と、当日更新ファイル `find <repo> -type f -newermt "<対象日> 00:00" ! -path "*/.git/*"`(decisions に未記載の作業を拾う)。
   - 新規ファイル(`??`)は中身の冒頭(docstring/見出し)を読んで何かを掴む。
   - decisions だけでは作業時間配分が分からないので、領域の比重は変更規模(diff stat)も見て判断する。

4. 本文(作業内容)を書く。下記「本文の書き方」に厳密に従う。

5. フィールドを JSON にまとめ、差し込みスクリプトを実行する:
   ```bash
   python3 ~/.claude/skills/work-report/fill_report.py --json /tmp/report.json
   ```
出力先(out)はユーザーの希望を聞く。既定はカレントディレクトリの `作業報告書_YYYYMMDD.xlsx`。

6. 生成したパスを伝える。本文もチャットに全文出して目視確認できるようにする。

## 本文の書き方(最重要)

このスタイルは固定。例外を作らない。

- である調。常体で統一する(です・ます調にしない)。
- 散文。箇条書き・見出し・記号(`#` `-` `*`)を一切使わない。段落だけで構成する。
- 内部表現を避ける。上長が読む前提で、社内用語・実装詳細を一般語に翻訳する。
- 構成: 冒頭の一文で「その日に取り組んだ領域」を概観し、以降の段落で領域ごとに詳述、 最後の段落で成果/今後の見通しを簡潔にまとめる。領域が複数あるときは「まず」「次に」「また」「最後に」で段落をつなぐ。

## フィールドの既定値

- company: InfoDeliver / name: 三浦由人 (テンプレ既定。変更指示があれば JSON で上書き)
- subject(件名): インターン
- progress(作業進捗状況): 途中
- next_plan(次回の作業予定): 続き
- special_notes(特記事項): 空。特筆事項があるときだけ書く。
- next_plan_date(次回作業予定日): 翌営業日が分かれば入れる。不明なら省略(空欄になる)。

JSON 例:
```json
{
  "work_date": "2026-06-19",
  "time_start": "13:30",
  "time_end": "19:00",
  "break_min": 0,
  "body": "2026年6月19日は、…(である調の散文)…",
  "next_plan_date": "2026-06-20",
  "out": "~/Downloads/作業報告書_20260619.xlsx"
}
```

## 完了の定義

- 必須(work_date / time_start / time_end / body)が埋まり、スクリプトが終了コード0で xlsx を出力した。
- 本文が上記スタイル(である調・散文・内部表現なし)を満たしている。
- 出力パスをユーザーに伝えた。
- ツール/ファイル読取りに失敗したら、止めずに何が取れなかったかを明示し、取れた範囲で下書きを出す。
