# throughline

この作業を貫く線を2本引く ── 動きの線(流れ: どう辿り着いたか)と、かたちの線(ロジック: 今どう動くか)。台帳(decisions)を翻訳し直すのではなく、源から人間語で起こす。どちらも「作る」より「更新」── ソースが変わったら作り直して上書きする派生ビュー。描画には触れず、データ/本文だけ差し替える。

名前の由来: いくつもの筋を貫く一本の線(throughline)=背骨。流れは時間の線、ロジックは構造の線。
(旧称 session-dag。図(DAG)レンダラを廃し対話ビューに作り替えた際に改名・両 view 対応に拡張。経緯 ../_dev/decisions.md。)

## 2枚の出力(両方を最新化する)

| | 流れ(動きの線) | ロジック(かたちの線) |
|--|--|--|
| 問い | どう辿り着いたか | 今どう動くか |
| 源 | この会話・作業(何が起きたか) | コード/決定(何が在るか) |
| 形 | 対話の beat(筋・詰まり→転回・誰の一手) | データの流れ・構造の図解 |
| template | templates/flow.html | templates/logic.html |
| 既定の出力先 | `.lavish/session-arc.html` | `.lavish/logic-<slug>.html` |

両方とも view≠真実(生成物)・単一HTML・file:// で開ける・描画には触れない。更新=ソースを見て作り直し上書き。

## 流れ(templates/flow.html)

1. この作業を筋(スレッド)に割る。一本道にしない。各筋は「何を作るか」「どう見せるか」のような線で、TODO 項目に対応するものと、どれにも乗らない脱線=「リスト外」に分ける。
2. 各筋を beat の列にする。種別4つ: start(きっかけ・気づき)/ concl(結論)/ wall(詰まり)/ resp(対応・転回)。直前の手を覆した resp は覆し(reversal)にする。意思決定の手には決定の印。
3. flow.html をコピーし、先頭の `THTAG`(筋見出し)と `BEATS`(beat 列)だけ書き換える。描画(CSS と `beat()`)には触れない。

BEATS の各 beat: `{th, who, k, t, decide?, rev?, ov?, now?}`。who = "me"(自分=あなた)/ "pair"(相棒)/ "out"(外部の相談役)、色で分かれる。k = start/concl/wall/resp。rev=true で同じ筋の直前 beat を覆した手・ov にその一行。直前の手と異なる筋の覆しは描けない(覆した相手の筋に置く)。

二層(流れだけ): live = `.lavish/session-arc.html`(上書き)。節目が閉じたら、語り(narrative)の置き場があれば `narrative/<date>_<slug>.html` に凍結 append(過去版は書き換えない)。

## ロジック(templates/logic.html)

1. 今のコード/設計を読み、「この仕組みは今こう動く」を html-deck の規律で削って1枚にする。状態の羅列でなく、仕組み(データの流れ・構造・原則)を描く。
2. logic.html をコピーし、本文(導入・データの流れの段・原則カード等)を差し替える。描画(CSS)には触れない。
3. 鮮度 = ソース(コード/決定)のハッシュをページに刻む。今のハッシュと違えば「古い」と表示し、作り直す。時刻でなく中身のハッシュで判定。

## language ルール(両方の合否点)

- 内部語を出さない。メタ語(spine/DAG/ledger/真実源 等)、id、スキーマ語、ツールの固有名、(できる限り)ファイル名を本文に入れない。読む人が知らない言葉はゼロ ── 流れは「その場にいた人間」、ロジックは「一般的なエンジニア」が分かる語だけ。
- 流れ = 人に話す言い方で一行(「〜と私が言った」)。詰まり→対応を落とさない(台帳は結論しか残さない・ここは転回が主役)。
- ロジック = 普通の言葉で「どう動くか」。実装の正は別(コード)にあり、これは生成した説明だと分かるように。
- 合否: 流れ=台帳を開かずに結論・詰まり→対応・誰の一手かを追えるか。ロジック=その分野の前提が無い人が今の動き方を掴めるか。追えなければ書き直す。

## 出力先(共通)

- lavish ワークスペース(`.lavish/`)があれば上表の既定先。
- 無く `成果物/` があれば `成果物/handoffs/throughline/<slug>-flow.html` と `-logic.html`。
- どちらも無ければ作業ディレクトリ直下に書き、置き場を確認する。
- slug は title から引いた短い英数。file:// で開けば動く(外部依存なし)。
