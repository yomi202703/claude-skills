## 2026-06-29 — 「不安だが問題は出ていない」の正体は開いたフィードバックループ

Context: 何度か使って問題は出ていないが不安が消えない、「他の不整合も検知できる方が良いのか」と owner。

整理した非対称: このスキルは W1 で送り手側・静的に徹する設計。ゆえに失敗は受領側(Windows)で、数日後、別の人の画面で表面化する。その失敗が送り手に返る経路は存在しない。よって「今のところ問題が出ていない」はほぼ情報を持たない — 出ていないのか、出たが届いていないのか区別不能。不安は神経質ではなく構造的に正しい信号。

「他の不整合」は2つの別物を指しうると切り分けた:
- (a) 同じ任務の中の本物の穴 — Mac→Windows ど真ん中なのに6軸が取りこぼしているもの。
- (b) 任務の外の一般的な整合性・正しさ — correctness バグ探し。

owner の選択(AskUserQuestion「緑が信用できない」): 範囲は今のまま、(a) を埋めて緑の精度を上げる。(b) は却下方向で合意。

## 2026-06-29 — (b) 一般的な不整合検知は入れない(won't do)

任務を「成果物の整合性全般」へ広げると、毎共有で軽く回せる今の良さが消え、deep-strict / code-review / factcheck と被る。SKILL.md 2行目「correctness バグ探しではない」を維持。owner も「緑が信用できない」を選び範囲拡大を選ばなかった。→ won't do。

## 2026-06-29 — A7「存在可否(Windows でファイルが在れるか)」を追加 ((a) の実装)

既存6軸の論理的な穴: A4 は中身の符号化を見るが、ファイルそのものが Windows に存在できるかを誰も見ていない。これはチェックアウトすら失敗する層で、緑でも静かに死ぬ筆頭。

A7 = 名前と実体の合法性(A4=中身 と対):
- 不正文字 `< > : " | ? *`
- 予約名 `CON`/`PRN`/`AUX`/`NUL`/`COM1..9`/`LPT1..9`(`aux.py` 等、拡張子付きでも不可)
- 末尾のドット・空白
- 大小のみ違う名前の衝突(Mac 既定で大小無視 → Windows で片方消失)
- MAX_PATH 260 超の深いネスト
- symlink(Windows で壊れ/実体化)

確信ゲート(W5)上の位置: 検出はするが、改名先・再構成・symlink の解決(コピー化か維持か)は意図依存 → Phase 3 の「要判断/提示」へ。勝手にリネームしない。Phase 4 に再 `ls`/`find` での不在再確認を追加。

変更箇所: SKILL.md Phase 1(A7 追記)、Phase 3(要判断に改名/再構成/symlink追記)、Phase 4(A7 再確認追記)。最小・局所。

## 2026-06-29 — ChatGPT 第三者チェック(chatgpt-web/nav.py consult)

owner「あと一応」で、上記の設計判断を ChatGPT に当てて検証。SKILL.md + decisions を実際に読ませ、「枠内で7軸が取りこぼす失敗モード／A7の置き方／範囲を広げない判断」の3点を聞いた。

結果 — 3点とも支持:
- 範囲を広げない: 妥当。「correctness バグ探しではない」を守れ、崩すと緑の意味が壊れる。
- A7 の置き方(検出する/改名は意図依存で要判断/確信ゲート不越え): 「かなり妥当」。A4(中身)↔A7(実体)の対が綺麗、static で W1 不抵触、自動改名しないのは import/README/参照を壊すから正しい。
- 穴は「方向」ではなく「チェックリストの端が少し欠けている」だけ、という総括。

挙がった枠内の取りこぼし3つ:
1. Mac メタデータ漏洩: ノイズ・バケツが `.DS_Store` しか名指さず、`__MACOSX/`・`._*`(AppleDouble)・`.AppleDouble/` が未記載。→ 確信あり・枠内・小。直した。
2. アーカイブ内部: zip/Drive 共有時に A4/A5/A7 が zip を binary 塊で止め、中のファイルに届かない。→ deferred(下記)。
3. A7 合法性リストの厳密化: 粒度の話。→ 保留。

## 2026-06-29 — Mac メタデータ漏洩を W2/W3 に明記(consult #1 を適用)

`.DS_Store` だけでなく `._*`(AppleDouble リソースフォーク)・`__MACOSX/`・`.AppleDouble/` を Mac メタデータとして不変の除外 = ノイズに格上げ。これらは常にノイズで配布物になり得ず、`.DS_Store` と同類 → 確信ゲートを越える(自動除外)。zip 共有で Mac の zip が `__MACOSX/`・`._*` を同梱しがちな点も W3 に注記。「緑なのに静かに紛れる漏洩」= owner の不安に直撃するため即適用。変更箇所: W2、W3 ノイズ・バケツ。

## 2026-06-29 — アーカイブ内部の検査は deferred(consult #2)

zip/Drive で渡すとき A4(符号化)/A5(改行)/A7(名前合法性)は今アーカイブを binary として扱い、内部ファイルに届いていない。A5 に「zip は .gitattributes が効かないので実バイト確認」の記述はあるが、内部ファイル一つ一つへの検査展開は未対応。
Deferred — トリガー: [zip/アーカイブ同梱での共有が実際に案件で発生したら]。実装すると毎共有のコストが上がるため、その形態が現実に来るまで入れない。配布形態がディレクトリ/個別ファイルのうちは穴にならない。
