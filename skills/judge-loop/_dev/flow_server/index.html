前進(フェーズ順序) 螺旋(戻り・差し戻し) routing(→ sub-skill) ★ = 単位発見(最重要)・grill-me/review-server へはファンイン + grill-me はあらゆる fork から / task-handoff は任意の分岐点から

不変ゲート G1–G9(全フェーズに常時かかる・継承拒否の番人)

DAG の各ノードを横断する制約。フェーズでなく常時。
G1 判定単位を継承しない・目的から導く
G2 決定論ファースト・LLMは意味の残差・確信度はコード導出(自己申告禁止)
G3 本番judgeは報酬でない・クローン禁止・holdoutで測る・報酬はオーナー反応
G4 主張を現実に合わせる・証拠を引いて検証可能に(断定/2値/隔離は P1 の選択)
G5 基準変更(carve-out)→全件 re-sweep + 俯瞰で巻き添え検出(検出のみ・救済 P4)
G6 grill-hook 却下とオーナー裁定を理由つき台帳へ(蒸し返し防止)
G7 インフラは単位依存度で時を決める: 単位非依存(scaffold/取り込み/アクセス層)=要件前OK / 単位依存=SETTLE後 / 例外=診断計器は config 駆動で早く・判別不能=pre-SETTLE=計器以外を遅らせる ★今セッションで二分法から書き直し
G8 基準を疑う前にデータを疑う(誤検知→証拠経路 / 抜け→原典被覆)
G9 過学習禁止・validity≠detectability・holdout/新規反応で汎化確認

毎ターン / 足場 / 中核手続き

毎ターン(5)

  • 1 フェーズ検出 — P0.0→P0u→P1→P2→P3→P4(螺旋・行進でない)
  • 2 ゲート強制 — G1–G9・opt-out 拒否
  • 3 routing — 所有 sub-skill へ・再実装しない
  • 4 出力命名 — 成果物と置き場所を名指す
  • 5 grill-hook 発火 — 次のオーナー判断・先回りしない

Premises(標準前提)

  • GT は遅れて来る — 確立前に end-to-end で回す・サーバーを早く
  • 本番judge = ローカル/開 Gemma級を API 配信
  • Day-1 scaffold — ディレクトリ + 四役 working-docs
  • 共有中立ファクト層 — 事実を1回中立抽出し全 judge が消費

Cores(手続き)

  • 単位スパイク — 分布が見える数だけ出力→反応→俯瞰診断
  • 俯瞰診断 — 構造共有FP→単位 / 散発際→基準 / 揺れ→ノイズ
  • SETTLE — unit-keying タグ K round 連続ゼロ→単位確定
  • 幻のリグレッション — A/A 床を測ってから A/B・境界は安全側
  • 1体制所見 ≠ N フラグ

合成(routing 先)

  • fork → grill-me / judge プロンプト → gemma-prompt
  • 共有 review サーバー(診断+GT作成+評価) → review-server
  • repo メモリ → claude-md(Scaffold)
  • 独立並行 main への fan-out → task-handoff
judge-loop の自前コードは K-run/flutter/分布diff ハーネスのみ。他は routing。
