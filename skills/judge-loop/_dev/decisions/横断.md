## 2026-06-24 — レビュー面の form factor: 診断=IDEエルゴ / blind=最小(非IDE)
- 発端(オーナー): 「人間が確認する時、基本 IDE 形式が理想かも」。
- 詰めの帰結: form factor はモードで割れる。診断面はメンタルモデルがコードレビューそのもの(多数ユニットを送る・evidence=go-to-definition・divergence=Problems・A/A·A/B·flutter=diff)→ IDE エルゴ(3ペイン+キーバインド+diff+Problemsパネル)が理想。blind GT作成面は逆=ファイアウォールが「情報の貧しさ」を要求するので意図的に最小(IDE の show-everything は blindness と戦う)。偶然でなく、ファイアウォールと観客が両モードを反対方向へ引く。
- 決定: 診断面は IDE エルゴ寄せ可・blind は最小のまま。本物の IDE 統合(単位=開発者エディタのファイル)は別 deferred=単位 settle まで建てない(G7)。
- なぜゲートでなく選択/方向か: 観客と防火壁から演繹した設計方向であって不変条件ではない。焼き込まず方向として残す。
- 反映: review-server SKILL.md W2(form factor 一行)・judge-loop TODO Deferred(本物のIDE統合・トリガ=単位settle&人間GT量)。
